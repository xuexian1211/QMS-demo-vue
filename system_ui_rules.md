1.列表页采用三部分内容，分别是：
1.1.工具栏，标签容器是div：这个div到上面的padding=10px，class=toolbar，其下不要出现标题。工具栏按钮包括：新增，编辑，查看，删除，导出，刷新。
1.2.搜索条件，标签容器是a-card，class=card，formitem之间要紧凑一些。
1.3.表格列表，标签容易是div，class=table-container，列表的行高要优化一下，紧凑一点，用户需要看到尽可能多的数据

2 列表页详细要求：
2.1 工具栏：包含「新增、编辑、查看、删除、导出、刷新」；列表支持多选框；
2.2交互规则：
① 编辑 / 查看：仅选中 1 条数据时可点击，否则不可点击；删除：多选后可点击，点击后需弹窗确认；
② 单据编号 / 行内查看 / 行内编辑：点击后在 content 区域打开对应页面（非弹窗、非 navbar 新标签）；
③ 工具栏编辑 / 查看：选中 1 条数据后点击，在页面区域打开对应页面（非弹窗、非 navbar 新标签）；
布局：整体紧凑，tags 标签与工具栏间无留白；列表行高优化。

3 表单页详细要求：（新增 / 编辑 / 查看）
3.1 结构：主要信息直显，次要信息 / 一对多数据用 tab 页签；查看 / 编辑页需携带对应单据 ID 的完整数据；
3.2 交互规则：
① 新增：按钮文字为「新增」，点击后在 content 区域打开空白表单页；
② 查看 / 编辑：打开后撑满整个 content 区域（非弹窗），不阻断 navbar / 菜单操作；
③ 返回按钮：查看 / 编辑页统一使用「详情页式返回」（点击后回到列表页，交互逻辑一致）；
④ 状态展示：表单页 H2 标题旁增加 “单据状态” 显示框；
⑤ tab 页签：修复编辑页 tab 失效问题，统一样式；
