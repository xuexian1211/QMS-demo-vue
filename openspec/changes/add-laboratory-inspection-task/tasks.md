# 实验室送检及管理模块 - 开发实施清单

- [x] 1. 扩展现有数据模型：
  - 更新 `qms_lab_task` 及 `qms_lab_result`，加入 `isBlindTest`（是否盲测） 及 `isUrgent` 字段。
  - 新建 `LabSample` 接口支持留样台账库位、保存期限跟踪。
  - 新建 `LabReport` 接口管理和集中检索实验室检测报告。
- [ ] 2. 完善触发联动的后端业务逻辑：
  - IQC/IPQC 检测时若遇到"实验室"必检项，自动创建 `PENDING_RECEIVE`（待接收）状态任务单及关联样品条码信息。
  - 提供 APQP/8D 手工触发检测任务接口，支持源单据的强绑定。
- [x] 3. 开发及优化前端工作台页面（送检任务工作台）：
  - 支持多任务总览、扫码接样。
  - 内置实验室优先级标记以及 `加急` 可视化标红展示。
- [x] 4. 开发内部实验登记与盲测控制功能页面：
  - 对没有 IoT 对接的实验实行界面人工填写，如果开启盲测 `isBlindTest=true` 必须屏蔽同物料其他任务结果历史数据。
- [x] 5. 开发委外检测管理页面：
  - 支持供应商选择、快递信息记录追踪及外部回传附件归档。
- [x] 6. 开发"留样管理台账"与"报告库"页面：
  - 留样页面提供库位搜索，批量勾选"销毁"/"退还"机制。
  - 报告库页面支持在线预览/下载 PDF 格式化后的报告文件。
- [ ] 7. 开发定时预警和触发反馈节点服务：
  - 每分钟扫描定时任务表，针对超 4 小时(加急)及 24 小时(常规)没生成报告的任务向钉钉发送报警信息。
  - 检测结果（含委托单据号及判定）反馈回主数据源关联任务。
- [ ] 8. 实施"致命缺陷"异常互锁：
  - 服务端订阅判定接口，一旦测定 `FAIL` 且原因属致命缺陷（如有毒超标），向 WMS 锁定在库该批次，并推品质总监。
