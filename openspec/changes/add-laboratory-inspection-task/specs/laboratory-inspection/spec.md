# 实验室检测管理特性规格

## ADDED Requirements

### Requirement: 实验室任务触发
系统 MUST 支持在检验触发链路中自动生成实验室检测任务，或者允许用户手动创建。
#### Scenario: 方案配置包含实验室项目
  - Given 系统中有一个已发布的检验方案，其中某检验项的 `isLabRequired` 为 `true`
  - When WMS 触发该物料的 IQC 检验并成功匹配此方案
  - Then 除了生成 IQC 检验任务外，系统应同步生成状态为“待接收”的实验室检测任务，并强关联原始单据。

#### Scenario: 手动触发实验室任务
  - Given 品质工程师处于 8D 处理流程中
  - When 工程师点击“新建实验室委托”，输入批次号与检验项目
  - Then 系统生成一条独立实验室任务，且来源标识为“8D人工委托”。

### Requirement: 样品管理与留样
所有转移至实验室的实物 MUST 具有唯一的追踪标识并管控留样周期。
#### Scenario: 样品接收与标定
  - Given 实验室任务处于“待接收”状态
  - When 实验员在界面操作“接样”并确认
  - Then 系统自动分配生成形如 `SPL-20260302-0001` 的样品唯一码，并在检测完成后倒推记录 3 个月的留样到期日期。

### Requirement: 双轨制分发
任务 MUST 可分配至内部检测能力节点或委外给第三方。
#### Scenario: 触发委外流程
  - Given 实验员判断无相关检测能力（如需 SGS 权威认证）
  - When 选择“委外检测”处理方式
  - Then 任务状态变更为“委外中”，并调用后端通知中心发送钉钉消息给采购部。

### Requirement: 自动化采集与报告
检测数值记录 MUST 由系统结合规格限进行防呆比对，并能导出报告。
#### Scenario: 数据自动判定
  - Given 一项拉力测试记录的目标公差为 `100 ± 5 N`
  - When IoT 设备或者人工录入实测值 `92 N`
  - Then 系统自动判定结论为 `FAIL`，高亮显示超出下限。
  - And 任务完成后，系统自动合并模板生成《实验室检测报告(PDF)》。

### Requirement: 预警与互锁闭环
实验室严重问题 MUST 能干预现场与物流。
#### Scenario: 加急时效预警
  - Given 一个被标记为加急的实验室任务已接收
  - When 过去 4 小时后状态仍未流转至完成
  - Then 触发延时机制，向实验室主任的钉钉发送督办提醒。

#### Scenario: 结果回落与外部防呆
  - Given 检测结果为致命 Fail
  - When 实验室提交最终审核判定
  - Then 系统回写更新源头 IQC 任务为“检验不合格”
  - And 触发一条外围拦截消息发送给 WMS `[Action: LOCK_INVENTORY, Batch: B2026...]`。
