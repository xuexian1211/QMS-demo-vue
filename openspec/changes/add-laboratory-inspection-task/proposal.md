# 提案：增加实验室送检与管理机制

## 1. 业务需求概述 (Business Need)
基于需求方案及前期设计的检验模型，**实验室送检（Laboratory Testing）**模块作为高精度、专项理化分析的核心，其设计旨在确保原材料及产品符合特定技术标准，并提供自动化的数据存证。

核心业务诉求包含：
- **触发机制与分流**：支持从 IQC/IPQC/OQC 自动触发，以及客诉/APQP阶段手工创建任务并绑定源头单据。支持内测与委外检测的分流管理。
- **数据采集与自动化**：针对高频设备（CMM、光谱仪等）实施 IoT 联网自动抓取。增加**盲测控制 (Blind Test)**，在特定实验时隐藏他人历史数据以消除主观偏差。
- **样品与留样管理**：记录留样库位与期限，监控台账状态，并支持到期自动提醒处置。
- **报告自动化与存证**：收集采集数据自动套用模板生成报告，构建独立的报告查询与打印中心。
- **预警监控**：提供时效延误预警（常规 24H 延误；加急 4H 钉钉强提醒）和结果异常预警（致命缺陷锁定物料并报警）。

## 2. 影响范围 (Scope)
- **检验任务联动**：扩展已有检验任务执行逻辑，根据项目属性自动拆分实验室任务。
- **前端页面建设**：
  - 送检任务工作台
  - 内部实验登记
  - 委外检测管理
  - 留样管理台账（新增）
  - 检测/试验报告库（新增）
- **预警与集成**：与 WMS/MES 单据交互互锁、钉钉消息推送通知、设备 IoT 数据采集接口。

## 3. 验收标准 (Acceptance Criteria)
- 当检验任务所关联的方案包含标识了“实验室”的检验项时，需自动生成关联的实验室检测任务并同步源单据信息。
- 内部实验登记支持盲测模式（隐藏历史记录）。
- 支持完整的留样生命周期管理，从存入库位到预定销毁日期的监控及到期提醒。
- 自动生成标准化检测报告，并能在“报告库”中集中查询与审批。
- 若结果为致命缺陷，系统立刻触发钉钉报警给品质总监，并向外围系统发送锁定指令。
- 超时未出报告（常规24H，加急4H）能准确触发延误标签及通知。
