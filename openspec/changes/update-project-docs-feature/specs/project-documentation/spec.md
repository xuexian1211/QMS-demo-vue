# 项目文档

## OVERVIEW
项目文档功能在 QMS 系统内提供了一个集中的存储库，用于查看和下载产品以及业务需求文档（BRD 和 PRD）。它支持高级渲染功能（包含 Mermaid 图表展现），并允许直接导出为 MD 和 PDF 格式文件。

## ADDED Requirements

### Requirement: 项目文档侧边栏
系统"必须"显示一个标题为 "QMS-Docs" 的左侧边栏，其中包含文档数据目录的层级树，并正确关联并映射到底层的文件目录 `/QMS-Docs` 及其下的 BRD 和 PRD 子文件夹。

#### Scenario: 查看可用的文档
**条件（GIVEN）** 用户在主界面上方和左边菜单栏导航到并点击 "项目文档" 菜单
**当（WHEN）** 页面加载并绘制侧边栏组件时
**那么（THEN）** 侧边栏的顶端显示系统根标题 "QMS-Docs"
**并且（AND）** 目录树中正确显示包含底层各 markdown 或者 txt 文件的 `QMS-BRD` 文件夹和 `QMS-PRD` 文件夹。

### Requirement: 引入 Mermaid 的高级 Markdown 渲染功能
文档详情查看器"必须"支持标准的普通 markdown 渲染结构，并能够特别渲染解析那些位于 markdown mermaid 代码块内所定义的 Mermaid 图表描述信息。

#### Scenario: 查看包含 Mermaid 图表的文档数据
**条件（GIVEN）** 用户点击选择并且点开了一个内部包含特定格式 mermaid 代码块的 markdown 格式的文档
**当（WHEN）** 文档的整体排版和内容在右侧视图被解析并渲染时
**那么（THEN）** 所有的普通文本段落和代码块的内容被正确格式化并且排列显示
**并且（AND）** 该特定格式的 mermaid 代码块被系统可视化引擎成功渲染呈现为一个具体的 SVG 图片图表，而不是依然显示为普通的文本或原始代码块形式。

### Requirement: 业务文档内容的资源导出模式 (MD / PDF)
用于浏览项目资料的文档查看器"必须"提供专门的文件保存并下载的功能集合，进而允许相关业务人员用户将当前系统正在浏览展示状态下的这部分文档提取出来并下载保存为系统的本地文件格式，比如最原始的文本格式 Markdown (`.md`) 文件，或者是便携式只读格式文档格式 (`.pdf`) 文件等跨平台的文档数据。

#### Scenario: 导出保存为 Markdown
**条件（GIVEN）** 用户的界面当前正在停留在查看某一特定的非空详情文档页面视图
**当（WHEN）** 用户移动鼠标或手势到右上角并主动并且成功点击系统功能界面按钮列表当中的 "下载 MD" 触发该功能逻辑的瞬间
**那么（THEN）** 用户端操作系统或者是相应的网络浏览器会直接并且主动地触发对应格式的文件数据下载，同时下载内容本身并且该下载的文件对象必然包含着与系统当前所展示的该系统文档完全一致的、准确无误的最核心最基础的原始内部所有的 markdown 文本文档字符段落信息以及内容数据段。

#### Scenario: 将信息导出为 PDF 格式进行归档保存
**条件（GIVEN）** 操作人员在系统使用过程中正沉浸在查阅项目的某一个指定的详情规格文档
**当（WHEN）** 操作人员完成相应的阅读需求，点击页面内预设用来辅助归档操作或者提供文档的离线存储特性的 "下载 PDF" 这一具体的界面按键之后
**那么（THEN）** 运行该系统的使用者的对应的这一个浏览器，将会主动调用相应处理逻辑，唤起浏览器的内置标准系统 PDF 页面打印或相关页面文件提取与保存生成生成特性的界面对话框
**并且（AND）** 借由这一功能处理后保存并产生导出来的这份只读 PDF 电子实体文档内部应当被确认绝对并且必然包含有能够等同映射原本系统所展示的内容本身，其中还能够把这些信息排版按照当前排布结构，甚至还包含里面一切已经被解析过的所有的已被呈现出来的特定形式的图片以及已经被渲染绘制完成显示出来的各种形态和类型特征的 mermaid 代码形式的所生成的各个形式图表图案等等所有的展示对象信息。
