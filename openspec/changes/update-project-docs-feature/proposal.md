# 更新项目文档功能

## 概要
将现有的“系统更新日志”（需求文档 PRD）功能转变为全面的“项目文档”模块。这包括更新菜单名称，更改侧边栏目录树的根节点，分离 BRD 和 PRD 目录，支持 Mermaid 图表渲染，并支持以 MD 和 PDF 格式下载文档。

## 动机
当前的“更新日志”页面范围有限，仅仅从 `PRD` 目录拉取 markdown 文件。用户需要一种更结构化的方法，在 `QMS-Docs` 目录下分离 BRD 和 PRD，需要高级渲染功能（Mermaid 流程图/图表支持），以及离线阅读功能（Markdown 和 PDF 导出）。

## 提案变更
1. **菜单和路由更新**: 
   - 将菜单项重命名为 `项目文档`。
   - 更新路由名称以反映 `project-docs`（项目文档）而不是系统更新日志。
2. **目录结构和数据源**:
   - 将数据源目录从 `/PRD` 更改为 `/QMS-Docs`。
   - 在侧边栏树中的根节点 `QMS-Docs` 下，为 `QMS-BRD` 和 `QMS-PRD` 提供可视化分组。
3. **高级 Markdown 渲染**:
   - 在现有的 `marked` 解析器旁边集成 Mermaid 渲染插件或逻辑。
4. **文档导出**:
   - 在内容区域添加下载按钮组（MD, PDF）。
   - 支持导出为 `.md`（原始下载）。
   - 支持导出为 `.pdf`（使用浏览器打印或客户端的 pdf 库，如 `html2pdf.js` / `jspdf`）。

## 风险与缓解措施
- 客户端的 PDF 生成可能会出现样式问题。缓解措施：使用可靠的打印样式表或 PDF 库，并在复杂的 markdown 表格/图表上进行充分测试。
- 如果不进行迁移，从 `PRD` 更改为 `QMS-Docs` 目录可能会使现有文件成为孤立文件。缓解措施：确保创建 `QMS-Docs` 目录并移动文件，或者在过渡期间使功能向后兼容。
