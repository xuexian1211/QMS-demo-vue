## ADDED Requirements
### Requirement: PLM质量文档分类管理集成
系统MUST支持接收来自PLM系统的质量相关文档，并按照业务场景进行结构化的二级分类管理。该分类需涵盖体系策划与基础管理、产品设计与开发、生产过程与现场执行、质量检验与监控记录、异常管理与持续改进五大一级分类场景及其下的各二级文档模块。

#### Scenario: 浏览由PLM集成的联合文档树
- **WHEN** 用户进入受控文档管理界面
- **THEN** 页面左侧的类型树自动呈现包括（质量手册、程序文件、管理制度、产品图纸、3D模型、BOM等）完整的双级（业务场景-文档模块）目录结构。

### Requirement: PLM与QMS双向文档标识支持
系统MUST在文档条目中明确标识文档的“主存”来源。对于PLM系统作为主存的文档（如：质量手册、程序文件、产品图纸、FMEA等），系统应支持单向读取或在线归档查询；对于QMS作为主存的文档（如：检验标准库、各类检验报告、问题解决记录等），应提供全生命周期的管理操作（录入、审批、作废），并根据需要进行状态反向同步。

#### Scenario: 查看核心工程图档信息
- **WHEN** 用户点击文档列表里的某项“产品图纸”或“3D模型”
- **THEN** 系统展示来源于PLM的在线图纸或技术规范，并在属性区标识其获取来源为“PLM主存”。
