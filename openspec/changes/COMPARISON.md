# 提案对比说明

## 旧提案：add-inspection-master-data-missing-functions

**范围**：仅关注补充缺失的功能

**包含内容**：
1. 缺陷分类独立管理页面
2. 量检具类型独立管理页面
3. 物料检验规格管理
4. 检验模板管理
5. 关联知识库管理（现象-原因、原因-FMEA）

**优点**：
- 目标明确，范围清晰
- 专注于补充缺失功能

**缺点**：
- **未包含您要求的功能**：不良现象、不良原因、物料档案、工艺路线、检验方案
- 未参照JNPF模板的页面结构
- 未统一现有的页面布局和交互模式

## 新提案：refactor-inspection-master-data-phase-1

**范围**：分阶段实施，第一阶段全面重构基础数据

**包含内容（第一阶段）**：

### 模块1: 基础数据重构
1. 不良现象管理重构
2. 不良原因管理重构
3. 物料档案管理重构
4. 工艺路线管理重构
5. 检验项目管理重构

### 模块2: 缺失功能补充
1. 缺陷分类独立管理页面
2. 量检具类型独立管理页面
3. 物料检验规格管理
4. 检验模板管理

### 模块3: 检验方案优化
1. 检验方案列表重构
2. 检验方案编辑优化

**优点**：
- ✅ **完全覆盖您要求的所有功能**
- ✅ 参照JNPF模板优化页面结构和交互
- ✅ 统一的布局模式和组件规范
- ✅ 分阶段实施，风险可控
- ✅ 第一阶段完成后可以进行第二阶段（知识库关联等）

**设计原则**：
- 保持现有技术栈（Vue 3 + TypeScript + Ant Design Vue）
- 参照JNPF模板的设计模式和布局结构
- 统一的页面布局：工具栏 + 搜索表单 + 数据表格
- 统一的编辑页面布局：返回按钮 + 标题 + 表单 + Tab页

## 两个提案的主要区别

| 对比项 | 旧提案 | 新提案 |
|-------|--------|--------|
| **实施策略** | 一次性补充缺失功能 | 分阶段重构和补充 |
| **范围** | 5个缺失功能 | 12个功能（5个重构 + 4个新增 + 3个优化）|
| **是否包含您要求的功能** | ❌ 不包含 | ✅ 完全包含 |
| **是否参照JNPF模板** | ❌ 不参照 | ✅ 严格参照 |
| **是否统一页面布局** | ❌ 不统一 | ✅ 完全统一 |
| **是否有第二阶段规划** | ❌ 无 | ✅ 有清晰规划 |
