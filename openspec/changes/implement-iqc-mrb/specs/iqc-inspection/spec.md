## ADDED Requirements

### Requirement: IQC 任务匹配与自动生单
系统 MUST 在接收到 WMS/SRM 的基于“生产指令卡”的收货报检信号后，基于 `OrgId`、`物料编码` 及业务类型自动拦截并匹配生效的检验计划模型，且当存在多条策略时必须高优先执行 P 级最高的计划（如：新供应商特殊计划）。

#### Scenario: 收到 SRM 报检基于优先级生成任务
- **WHEN** 系统接收到供应商 S01 的 M001 物料生单信号，存在“常规抽检 P20”与“前三批次严检 P5”双计划
- **THEN** 系统 MUST 选择执行 P5 加严抽样计划
- **THEN** 生成的任务 MUST 获取计划中所约定的“本地化覆盖公差”而并非原始全量模板参数

### Requirement: 动态抽样与特性指引
系统 MUST 结合当批次来料总数量及关联的 AQL 抽样规则，自动计算出样本量 (n)、允收数 (Ac) 和拒收数 (Re)，并向检验员清晰展示各检测项所属的重要度分类（Major/CC/SC）。

#### Scenario: 基于数量触发动态 AQL 计算
- **WHEN** 检验员打开数量为 5000 件的来料任务，且预设 AQL 0.65 Level-II 时
- **THEN** 系统 MUST 在界面显示该批次判定标准为：抽样数 n=200，Ac=3，Re=4
- **THEN** 所有与关键安全特征 (CC) 绑定的检验项 MUST 展现强制拦截标识

### Requirement: 实测结果防呆与自动判定拦截
系统 MUST 在每个包含数字上下公差的项目行中，自动且实时地判别质检员录入的“实测值”是否在合规区间，对超出部分直接锁定为不合格，并阻止不选择原因的流转。

#### Scenario: 超差数据录入并强制要求现象描述
- **WHEN** 检验员在宽限要求 10.0 (+0.1/-0.1) 的项目录入 `10.5`
- **THEN** 该行 MUST 立即且不可篡改地由前端判定为 `FAIL` 标签
- **THEN** 用户尝试直接点“提交”时，系统 MUST 提示并强制要求对这行 `FAIL` 选择预设在字典中的 “不良现象”（如：尺寸超差 DP-001）方可继续

### Requirement: 库存联动与实验触发闭环
系统 MUST 对总体评定合格 (`QUALIFIED`) 的任务向上游 WMS 广播解锁指令；凡打上“实验室送检 (LAB)”标签的子条目 MUST 发出实验室挂载任务。

#### Scenario: 检验合格的清关联动
- **WHEN** 任务提交为 `QUALIFIED` 结论时
- **THEN** 该结果 MUST 通过设定的事件 API WebHook 将指令发向 WMS 或 SRM 同步更新状态“允许放行入库”
- **THEN** 当前任务包含 LAB 送检标记的项目 MUST 由系统生出并流转至后续化验室实验排队序列
