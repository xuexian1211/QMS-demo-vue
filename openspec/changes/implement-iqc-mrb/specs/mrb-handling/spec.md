## ADDED Requirements

### Requirement: 基于失败检验的端到端触发
系统 MUST 在 IQC 层面检测总体结果为 `UNQUALIFIED` 或达到一定程度致命严重缺陷预警 (Major/CC) 时，静默捕获这一信号并自动产生一份《来料不合格品审理单》（MRB）推送至质量工程师 (SQE) 待办。

#### Scenario: IQC不合格引发的自动报警与物料标定
- **WHEN** 对批次记录编号为 `BATCH001` 的进料主单终审被确认为异常
- **THEN** 生成相关异常处理号 `MRB-[IQC任务流水号]` 的审单，并将状态设定为 `DRAFT` 草稿待决模式
- **THEN** 系统 MUST 即时触发 API 给现场 IQC 检验员下发消息挂置要求，对现场实物强制张贴红色（不合格）或黄色（待定）标识加以视觉防呆隔离

### Requirement: 复杂组织的在线协同评审机制
系统 MUST 构建基于多角色协同（SQE、采购、生产、技术）的会签流；并能通过智能定级机制根据影响阈值调配不通过的最高审批人。

#### Scenario: 跨部门自动提醒与会签的流转操作
- **WHEN** MRB 牵头人将其流转为 `UNDER_REVIEW` (评审中)，指派生产部与采购部对当前缺陷处理发表评估意向
- **THEN** 系统 MUST 调用配置在系统的第三方 IM (如钉钉、飞书) 的消息机器人下发会签审批请求链接
- **THEN** 所有意见 MUST 被收集至审理单内，任何一人未点批准前无法终审 `APPROVED` 及结案

### Requirement: 处置分支与 COQ （外失效成本）挂靠
对于最后 MRB 结案状态的判定，系统 MUST 规范化可选项，禁止手填不明含义结果；同时需要能将这些决定衍射至质量成本看板。

#### Scenario: 不良返修处理下衍生成本与返工单
- **WHEN** 会签结束后终审确认选用“返修/挑选 (REWORK)”这一处置方式
- **THEN** 审理单 MUST 提供辅料成本估算与废工时表单接口收集外失与内失故障成本（COQ）
- **THEN** 合格退回品复入库流程需向 WMS 请求部分解除锁定信号

### Requirement: 强制启动 8D 整改与防止异常蔓延的知识沉淀
系统 MUST 为防止同样供应商问题再现支持与预防工具集成闭环，当指定或连续触发时要求强制 8D 单流转并在最后强制知识库收敛。

#### Scenario: 重复不良诱发供应商强制整改反馈链
- **WHEN** 同一供应商出现连续三个批次的 CAU-001 原因（来料公差/规格偏差）
- **THEN** 系统后端 MUST 强制阻止人工勾选直接“结案”
- **THEN** 系统 MUST 锁定要求先派生或挂载一条关联的 《8D 预防纠正回复单》，等待审批流程关闭后，且通过界面向“更新经验库检查表”打钩方可最终判定为 `CLOSED`
