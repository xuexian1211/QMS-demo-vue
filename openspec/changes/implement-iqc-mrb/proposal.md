# Change: 深度实现 IQC 来料检验执行与 MRB 进料不良异常处理闭环

## Why
目前系统仅具备生成基础 IQC 任务的骨架与手工处理流程。为了满足严谨的工业级质量管理及 IATF16949 的要求，需要与前端供应链系统（WMS/SRM）进行深度闭环集成。进料检验过程需要支持复杂的策略匹配、动态抽样计算以及对特性分类的精准管控；而进料不良 (MRB) 处理阶段则需要结构化的在线会签评审、标准化的处置分支，并能够无缝拉升至 8D 纠正预防流程以形成质量管理闭环。

## What Changes
- **IQC 业务触发与任务生成逻辑**：接入 WMS/SRM 收货报检信号，基于多维度匹配策略生成检验任务，支持优先级规则执行、方案快照关联及本地化参数覆盖。
- **检验执行与动态计算**：支持基于特性分类 (Major/CC/SC) 的指引；根据收货量及 AQL 动态计算样本与收拒线 (n, Ac, Re)；支持实验室送检关联；全链路实测校验防呆及不良现象强制拦截。
- **库存闭环与绩效回写**：合格后自动解锁上游库存；检验与异常结果自动关联供应商绩效评估系统。
- **MRB 在线评审流程**：IQC 不合格自动生单并触发 SQE 审查；打通组织协同（通过钉钉等 API）在线会签；基于风险级别进行不同管控流向的高层审批。
- **结构化处置分支**：严格限制退货(RTV)、挑选/返修、特采让步接收等标准化分支路径；系统自动推算内外部失效成本 (COQ)。
- **预防与改进机制**：严重问题强制触发供应商 8D 调查；结案提供更新 FMEA、控制计划与经验库等机制保障防再发。

- Affected code:

  - 异常管理业务视图 (`src/views/production-quality/exception-handling/`)
