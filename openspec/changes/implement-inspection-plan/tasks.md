# 检验计划管理任务列表 (Inspection Plan Tasks)

1. [ ] Mock 数据及类型定义：在 `src/types` 和 `mock` 文件中添加 `InspectionPlan` 及级联明细实体的接口和测试数据。
2. [ ] 路由及页面初始化：在 `src/router/` 和 `src/views/quality-master-data/` 中配置检验计划主页及详情页的路由映射。
3. [ ] 列表及检索组件实现：实现 `InspectionPlanList.vue`，集成高级状态过滤器、基于多选框的 `orgIds` 组织架构过滤器以及统一风格的数据表格。
4. [ ] 权限及重用隔离逻辑开发：实现前端只读保护守卫逻辑，针对集团级（`orgId` 匹配规则）或未授权的数据禁止显示行内编辑及删除入口。
5. [ ] 实现复制方案对话框组件：开发 `CopyInspectionPlanModal.vue` 组件，包含目标编码、目标名称及组织切换的字段收集，并连接至复制数据逻辑动作。
6. [ ] 详情页页签与基础信息开发：实现 `InspectionPlanDetail.vue` 容器及“基础信息”选项卡内容页的编辑/展示态自适应排版。
7. [ ] 模板加载与检验明细定制开发：在详情页中实现检验明细列表组件，响应“关联模板”变更并动态拉取映射结构，提供行内明细定制化调整（例如：调整公差规则、开启实验室测试标记）。
8. [ ] 生命周期流转交互：实现草稿至批准状态机涉及的 UI 流程向导交互（包括提交、冻结表单输入），并在列表中直观呈现状态标记。
9. [ ] 单元及集成自测：验证从集团模板复制至工厂级、随后在该工厂级方案上进行部分规格修改且互不污染的核心业务环路逻辑。
