# 优化基础数据管理 (Optimize Basic Data Management)

## 问题陈述 (Problem Statement)

当前基础数据模块（包括客户、供应商、物料和缺陷原因）的实现已经支持 QMS 所需的核心功能。但是，对照系统路线图（Roadmap），仍存在几个明显的差距：
1. **缺乏多组织架构支持**：基础数据实体目前是扁平化的全局数据，缺乏细粒度到 `orgId` 级别的租户或工厂隔离，这对于大型制造集团来说至关重要。
2. **缺失批量导入功能**：由于缺少 Excel 模板下载和批量导入功能，导致初始数据录入工作极其繁琐。
3. **层级数据的 UI 展示不佳**：具有层级结构的数据（如物料分类、缺陷原因）目前使用平铺列表或简单的下拉框展示。对于层级较深的数据，需要采用“左树右表”的高级布局结构。
4. **缺乏附件与资质管理**：客户和供应商档案无法存储资质文件（如营业执照、SQE准入证明等），也无法提供有效期预警功能。

## 解决方案 (Solution)

我们将在基础数据存储模型和 UI 筛选条件中引入多组织（Multi-Organization）字段，增加 Excel 批量导入功能，将层级数据页面的布局重构为“左树右表”结构，并在客户/供应商档案中集成资质文件的附件上传功能。

- 在客户（Customer）、供应商（Supplier）、物料（Material）和工艺路线（Process Route）的实体定义和 Mock 数据状态中添加 `orgId`（所属组织）字段。
- 在各列表页面的工具栏中引入 `导入 (Import)` 操作。
- 在客户和供应商的编辑表单弹窗中实现一个附件区域（包含上传组件）。
- 为具有深层级分类的物料（Material Product）和缺陷原因（Defect Cause）页面创建分栏布局（左侧显示树结构，右侧显示数据表格）。
