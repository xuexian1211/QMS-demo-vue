# 检验计划模型设计与实现任务 (Design Inspection Plan Tasks)

1. [x] 扩展及定义新检验计划接口类型：在 `src/types/` 和 `mock/` 中重构/新增 `InspectionPlan` 的 TypeScript 定义（集成触发器配置、参数覆盖结构映射等新设计）。
2. [x] 调整列表页展示项及列配置：升级计划列表页面使其展现 `plan_status`、`is_latest_version`、触发类型及责任人等核心字段属性。
3. [x] 版本管理与状态机实现：开发支持在 UI 端实现"发布新版本"、"作废旧版"、"挂起"等操作交互动作。
4. [x] 开发动态触发配置表单(Trigger UI)：为计划新增基于事件(如选定采购接收入库选项)及时间/数量(Cron频次)选择表单。
5. [x] 扩展与方案继承组件：结合之前开发的复制模态框，实现集团模板向下直接继承、允许局部重载(Edit Copies)业务。
6. [x] 动态参数覆盖界面(Target Overrides)：在关联方案展开明细后，提供供修改特定项公差或检查工具等的小型行内编辑器界面，产生参数覆盖JSON。
7. [x] 快速响应级别配置项：界面勾选新增锁定规则的响应动作(如勾选发送至经理预警)。
8. [x] 单元/集成测试：使用新定义模型流转测试检验任务被正确下发或参数覆盖能顺利作用在衍生任务表单。
