# Change: 检验任务自动生成与模拟功能

## Why
在目前 QMS 系统中，检验方案（Scheme）和检验计划（Plan）的基础模型已完成。为实现 QMS 与外部系统（如WMS、MES、ERP等）的闭环，需要设计一套统一的基于业务事件触发的检验任务自动生成机制，并提供控制台进行逻辑联调与模拟模拟验证，保证数据准确传递与分配。

## What Changes
- 开放一个统一的业务事件接收接口 `POST /api/v1/inspection/task/trigger`，供外部系统在特定节点调用。
- 基于请求参数（组织、应用模块、业务单据信息与批次上下文等）和现有检验计划模型，完成检验任务实例的自动匹配与生成。
- 采用参数覆盖或特殊动态策略等，将任务按角色设定分派给检验员。
- 在系统设置或计划管理模块集成“触发模拟器”，支持输入 JSON 报文以在线检测判断链路图和模拟生成判定。
- 新增调用日志（Trigger Logs）展示外部系统调用情况与内部响应明细（成功、忽略、异常详细原因等）。

## Impact
- Affected specs: `inspection-task-generation` (新增)
- Affected code: 后端检验任务控制器与业务逻辑层、前端计划管理或系统设置中的触发器模拟页面与日志查看页。
