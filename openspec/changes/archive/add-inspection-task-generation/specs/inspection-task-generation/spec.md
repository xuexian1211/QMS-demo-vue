## ADDED Requirements
### Requirement: 检验任务生成统一接口
系统 SHALL 提供一个统一的 HTTP 接收接口，供外部系统在特定业务节点触发检验任务的生成操作。

#### Scenario: WMS 采购收货触发 IQC
- **WHEN** WMS 提交采购收货单，并发送含有 `org_id`、`inspect_type: IQC`、来源单据信息以及物料编码等数据的请求时
- **THEN** 系统接受该触发事件并记录接收日志，启动内部的匹配逻辑。

#### Scenario: MES 生产指令触发 IPQC
- **WHEN** MES 下达生产指令，请求 `inspect_type: IPQC` 并附加扩展上下文时
- **THEN** 系统捕捉业务动作，按扩展维度寻找适配的过程检验计划。

#### Scenario: 金蝶 ERP 发货审核触发 OQC
- **WHEN** 金蝶云星空触发销售发货单审核动作，请求 `inspect_type: OQC` 等字段
- **THEN** 系统应通过统一接口接收该发货检测请求，为生成成品检测任务准备数据。

### Requirement: 检验计划匹配与生成逻辑
系统 SHALL 在接收到触发请求后，根据组织、检验类型、物料及扩展字段进行维度检索；匹配到唯一适用或最高优先级计划后， SHALL 校验其触发配置并生成对应任务实例。

#### Scenario: 成功匹配并生成任务
- **WHEN** 系统找到适用计划，且触发事件满足如“单据一致”等预设要求时
- **THEN** 锁定方案快照，如果有参数覆盖配置则应用它，分配任务给计划默认对应人员，并记录状态为 Success 的日志。

#### Scenario: 动态策略由于条件不满足被跳过
- **WHEN** 当批次为第 4 批但适用策略要求“前 3 批生成检验”时
- **THEN** 系统不生成正式检验任务，日志内记录为 Ignored 并附上详细原因“当前批次为第 4 批，不满足触发条件”。

#### Scenario: 参数异常导致匹配失败
- **WHEN** 计划中存在缺失必须配置项（如物料规格未定义等异常情况）
- **THEN** 生成任务执行被中断，内部记入状态为 Failed 的请求日志。

### Requirement: 触发模拟器与执行链路诊断
系统后台 SHALL 提供一个专门的触发模拟器，用于在免正式生成数据的情况下，验证由外部传入 JSON 报文后系统内部的执行判定路径。

#### Scenario: 使用模拟器排查匹配规则
- **WHEN** 实施人员在模拟器贴入 MES 传送的 JSON 进行测试验证时
- **THEN** 系统不产生实际检验任务记录，而是返回包含“输入数据 -> 匹配到的计划 -> 条件校验判断情况 -> 模拟建议结果”节点的树状路况图与判断总结。

### Requirement: 异步日志与集成监控追踪
系统 SHALL 全量记录对统一触发接口的所有访问动作，将其归类为不同解析状态并供运维和质量专员溯源追踪。

#### Scenario: 查阅触发错误日志
- **WHEN** 用户通过监控台查阅失败请求时
- **THEN** 日志记录必须体现 请求时间、请求 ID、来源（系统与单号）、总判定结果及导致失败与回滚的根本原因文字提示。
