# 基础数据：客供关系映射功能 (Basic Data: Customer-Supplier Mapping)

## Context
在质量管理系统（QMS）中，客供关系映射（Customer-Supplier Relationship Mapping）是指对企业内部及外部供应链中，不同角色之间质量职责与数据交互的逻辑对齐。
目前系统在 `src/views/basic-data/` 目录下存在 `CustomerArchive.vue` 和 `SupplierArchive.vue`，以及一个混用的 `CustomerSupplier.vue`，这并不满足基于全生命周期防错和追溯的要求。
其核心意义在于打破“数据孤岛”，实现全价值链的质量协同，具体体现在四个维度：
1. 实现全链路质量追溯：建立“供应商-来料-生产过程-成品-客户”的映射关系，实现正向追踪和逆向溯源。
2. 统一质量语言与标准：将客户的特殊特性要求（CTQ）直接映射到供应商的检验标准中。
3. 驱动供应商绩效的数字化评价：支持动态优化合格供方清单和协同改进。
4. 支撑数字化转型与系统集成：为 QMS 与 MES 等系统集成时的物料、客供编码转换提供基础数据支撑。

## Proposed Change
重构现存的“客供合一 CRUD”，将其转化为“客供关系与追溯映射矩阵”管理模块。
该模块不仅管理客户和供应商实体的绑定（VMI/代工），更侧重于**数据和标准的链路映射**，包括：
1. 客户物料与客供系统内物料的编码映射。
2. 客户特殊特性（CTQ）与对应物料的检验模板映射。
3. 供应商供货物料的系统内部编码与旧系统/外部系统编码的映射。

## Capabilities
- `tracing-and-mapping`: 定义客户物料、外部系统物料与内部物料的互通映射及全链路质量协同的配置体系。
