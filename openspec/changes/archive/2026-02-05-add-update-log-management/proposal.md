# Change: 新增系统更新日志管理

## Why
随着 QMS 系统的持续迭代和功能增强,用户需要了解系统的更新内容和新功能。同时,开发团队需要在每次 OpenSpec 提案归档时自动记录更新内容,以便追踪系统演进历史。这个功能可以提高系统透明度,帮助用户快速了解系统变化,并为开发团队提供清晰的变更记录。

## What Changes
- 在顶部导航栏的消息通知按钮(🔔)旁边新增"更新日志"按钮(📝)
- 创建系统更新日志管理页面,包含:
  - 更新日志列表展示(支持分页、搜索、筛选)
  - 更新日志详情查看(包含版本号、更新时间、更新内容、影响范围等)
  - 更新日志创建和编辑功能(包含富文本编辑器)
  - 更新日志发布管理(草稿、已发布状态)
  - **功能设计文档关联**(每条更新日志可关联详细的设计文档)
- 创建功能设计文档管理功能,包含:
  - 支持按角色划分任务(前端开发、后端开发、测试)
  - 支持多文档关联(一条更新日志可关联多个设计文档)
  - 支持文档查看和导出(便于团队成员分别阅读)
  - 支持从 OpenSpec tasks.md 自动解析生成设计文档
- 在 OpenSpec 提案归档时自动触发更新日志生成(基于 proposal.md 和 tasks.md 内容)
- 支持手动创建更新日志条目和关联设计文档
- 更新日志通知机制(新更新日志发布时在消息通知中显示)

## Impact
- **新增功能模块**: 系统更新日志管理
- **受影响的前端文件**:
  - `src/layout/MainLayout.vue` - 顶部导航栏新增更新日志按钮
  - `src/views/system/` - 新增更新日志相关页面组件
  - `src/components/RichTextEditor.vue` - 新增富文本编辑器组件
  - `src/router/index.ts` - 新增路由配置
  - `src/types/index.ts` - 新增类型定义
  - `src/api/updateLog.ts` - 新增 API 服务层
- **受影响的后端**:
  - 使用 Mock 数据，不需要后端 API
  - 使用自动生成脚本替代后端集成
- **受影响的 OpenSpec 能力**:
  - `system-management` - 扩展系统管理能力
  - 新增 `update-log-management` 能力
- **用户体验提升**:
  - 用户可以快速了解系统的最新更新内容
  - 提高系统透明度和用户信任度
  - 减少用户对新功能的困惑
