# 提案：检验主数据第二阶段 - 关系与高级功能

## Why
在第一阶段建立的基础主数据模块之上，通过实现关键的业务逻辑关系（知识库）、数据互操作性（导入/导出）和高级可用性功能，进一步扩展系统能力。

## What Changes
1. **知识库（关系）**：
   - 建立不良现象与不良原因之间的关联。
   - 将检验项目映射到潜在的不良现象。
   - 为 FMEA 集成准备数据结构。

2. **数据操作**：
   - 实现 Excel 导入/导出功能，用于批量数据管理。
   - 增强批量操作（如状态变更）。

3. **高级搜索**：
   - 实现用于复杂查询的高级筛选组件。

4. **UI/UX 一致性**：
   - 统一详情/编辑页面的交互模式。
   - 复杂实体（如检验项目、不良原因、抽样方案）采用独立页面（参考工艺路线）。
   - 简单实体（如物料分类、客户/供应商）采用模态框详情（参考计量单位）。

## Design
- **关系**：使用关联表（在模拟数据中使用 ID 数组）来链接实体。
- **UI 交互**：在现有的编辑页面（抽屉/模态框）中添加“关联”选项卡，以便在不干扰主视图的情况下管理这些链接。
- **导入/导出**：使用通用工具函数处理 JSON <-> Excel/CSV 转换（如果不需要引入重型库，则使用模拟实现）。
