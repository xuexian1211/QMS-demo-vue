# 设计文档: 检验主数据重构 - 第一阶段

## 架构概述

本设计对S_QMS系统的检验主数据管理进行全面重构，参照JNPF框架的设计模式，保持现有技术栈（Vue 3 + TypeScript + Ant Design Vue）。

## 设计原则

### 1. 统一的页面布局

参照JNPF模板，所有列表页面采用统一布局结构：

```
┌─────────────────────────────────────────┐
│ 工具栏（新增、编辑、删除、导出等）    │
├─────────────────────────────────────────┤
│ 搜索表单（多个搜索条件）             │
├─────────────────────────────────────────┤
│ 数据表格（列展示、操作按钮）         │
└─────────────────────────────────────────┘
```

### 2. 统一的组件规范

采用Ant Design Vue组件库，保持一致的交互模式。

## 数据模型设计

所有数据模型包含 `orgId` 字段，支持多组织隔离。详细接口定义见各功能规格文档。

## 页面组件设计

### 新增页面组件

1. **DefectCategory.vue** - 缺陷分类独立管理
2. **MeasureType.vue** - 量检具类型独立管理
3. **MaterialSpec.vue** - 物料检验规格管理
4. **InspTemplateList.vue** - 检验模板列表
5. **InspTemplateEdit.vue** - 检验模板编辑

### 重构页面组件

1. **DefectPhenomenonList.vue** - 不良现象管理（拆分缺陷分类）
2. **DefectCauseList.vue** - 不良原因管理
3. **MaterialProduct.vue** - 物料档案管理
4. **ProcessRoute.vue** - 工艺路线管理
5. **InspectionItemList.vue** - 检验项目管理
6. **InspPlanList.vue** - 检验方案列表
7. **InspPlanEdit.vue** - 检验方案编辑

## 样式规范

统一的CSS类名和样式定义，详见源码。

1. 表格分页加载
2. 树状数据懒加载
