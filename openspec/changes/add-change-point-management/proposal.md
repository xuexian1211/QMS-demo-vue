# 提案：变化点管理

## 概述
本提案在 S_QMS 项目中引入“变化点管理”模块，遵循 BRD 中的详细要求和用户的初始请求。该模块管理 4M1E（人 Man、机 Machine、料 Material、法 Method、环 Environment、测 Measure）变化以预防质量问题，通过智能路由和 MES 互锁在风险控制与生产效率之间取得平衡。

## 动机
IATF16949 明确要求对变化进行管理。S_QMS 需要一个主动的系统，以确保在全面生产前对车间的任何变化进行评估和验证。该流程必须具备快速申报、精准拦截（锁机/锁工单）和闭环验证的功能，并与钉钉和 MES 系统深度融合。

## 拟定变更
1. **4M1E 结构化提报与智能定级**：
   - 实现针对人、机、料、法、环、测变化的结构化提报。
   - 利用风险矩阵引擎将风险分级（低、中、高）。
   - 低风险变化经班组长确认后自动通过；中/高风险变化触发 QE 或品质总监审批，并强制触发验证流程。

2. **生产互锁与试生产控制 (MES 互锁)**：
   - 对于中/高风险的变化，自动触发 MES 接口指令“锁定机台/工单”，禁止批量报工。
   - 生成特定的“试生产检验任务”（例如：连续首件检验合格）。
   - 只有在验证成功后，才向 MES 发送“解锁/放行”指令。

3. **闭环与知识沉淀**：
   - 当变化点成为常态后，联动更新相关标准文件（SOP、控制计划 CP、FMEA）。
   - 将处理案例同步沉淀至经验库，为未来相似变化提供智能推荐。

4. **UI/UX 实现（PC 端为主，PDA 为辅）**：
   - **PC 端 (Web 控制台为核心)**：实现全部功能。除了系统监控与配置（如变化点中央看板、风险矩阵与规则配置、变化点台账与追溯），还必须支持操作级功能，包括新增及审批变化点申报表单、制定并在 PC 端执行试生产验证任务。
   - **移动端 (PDA/手机 - 辅助现场)**：主要提供现场快捷录入和反馈能力，如一键申报页、现场验证任务单（IPQC 专属视图）、班组长快速审批列表等。

5. **预警机制**：
   - **高风险盲动预警（红色）**：如果在 QMS 锁定期间 MES 接口反馈受控机台有批量报工数据，系统立即通过钉钉触发预警推送给生产总监和品质总监。
   - **验证超时预警（橙色）**：如果锁机后超过 4 小时未完成试生产验证，触发预警。

## Spec 增量清单
- `change-point-management`：定义了提报、风险路由、MES 互锁、验证任务、知识同步、预警以及用户界面的具体需求。
