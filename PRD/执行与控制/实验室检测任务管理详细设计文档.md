# 实验室送检与检测管理 — 详细设计文档

> **版本**: V2.0
> **日期**: 2026-03-03
> **模块位置**: 执行与控制 > 实验室管理(Lab)

---

## 1. 概述

实验室送检（Laboratory Testing）模块作为高精度、专项理化分析的核心，其设计旨在确保原材料及产品符合特定技术标准，并提供自动化的数据存证。本模块覆盖从任务触发、样品接收、内部/委外检测执行、数据自动采集与判定、留样管理到报告归档的全闭环生命周期。

核心业务域：
1. **触发与分流** — 多源自动触发（IQC/IPQC/OQC联动）+ APQP/8D手动触发 + 内测/委外双轨分流
2. **数据采集** — IoT 设备自动抓取 + 手工录入 + **盲测模式**（消除主观偏差）
3. **样品管理** — 唯一标识、库位存放、留样到期监控与处置
4. **报告管理** — 自动生成报告、委外报告录入、集中查询与审核
5. **预警监控** — 时效超时预警（常规24H / 加急4H）+ 致命缺陷报警与互锁
6. **业务闭环** — 结果回写源头任务、WMS/MES 互锁控制

## 2. 业务触发机制详情

### 2.1 检验方案联动触发 (自动生成任务)
* **前置条件：**
  * 主数据模块中已配置并启用了相关的“实验室”属性（如开启 `isLabRequired`）。
  * IQC/IPQC/OQC 创建检验任务，并绑定包含上述检测项的检验方案。
* **业务逻辑描述：**
  * 在质量控制过程（如来料收货、生产过程巡检）中，当触发系统标准的 IQC / IPQC 等质量检验业务模块时：
    1. 系统引擎根据绑定方案，提取其中标记为“实验室检测项（`isLabRequired = true`）”的具体指标。
    2. 基于分离解耦的原则，主干检验模块只保留非实验室项的验证控制，而对实验室检测项，系统通过事件分发机制，向“实验室模块”发射一条“需送检任务单”的服务请求。
    3. 实验室后台监听该事件后，自动构建全新的 `qms_lab_task`（状态置为 `PENDING_RECEIVE` 待接收），并根据源检验属性复制物料、批次信息及关联的 `sourceTaskNo`（检验模块源单）。

### 2.2 手动创建与特定专项业务触发
* **前置条件：** 针对非标准化批量或客诉等偶发需求。
* **业务逻辑描述：**
  * **8D 控制流程触发**：系统客诉模块或产生外部反馈触发失效分析，品管工程师在系统中可直接下达 `MANUAL_8D`（8D 实验室委托）属性的任务。必须在此节点记录对应的 `sourceBillNo`（关联如 "8D-202x-..." 单号）。
  * **APQP 新品验证触发**：研发部门试产件属性核验环节，直接手动开单建批，标定源状态为 `MANUAL_APQP`，以走实验分析及材料确证通道。

## 8. 详细数据模型设计

### 8.1 主表：实验室任务表 (`qms_lab_task`)
**说明：** 记录每次送实验室检测的单一主体控制任务，并定义整体进度。
| 字段名称 | 类型 | 是否必填 | 说明/用途 |
|----------|------|---------|------------|
| `id` | VARCHAR(50) | YES | 物理主键标识 |
| `task_no` | VARCHAR(50) | YES | 实验室任务号（系统依序生成，如 LAB-YYYYMMDD-XXXX） |
| `source_type` | VARCHAR(20) | YES | 单据来源类型：`IQC/IPQC/FQC/MANUAL_APQP/MANUAL_8D` |
| `source_task_no` | VARCHAR(50) | NO | 源检验执行主单号引用 (如来料则为 TSK-IQC-...) |
| `source_bill_no` | VARCHAR(50) | NO | 第三方或关联表单引用（收料单、订单、8D工单） |
| `insp_item_code` | VARCHAR(100) | YES | 待检项目标准化对应主物主键 / Code (如：拉力特性分析) |
| `material_code` | VARCHAR(50) | YES | 监测物料规格标识名 |
| `batch_no` | VARCHAR(50) | YES | 流水批次，针对追溯互锁最核心的条件 |
| `status` | ENUM | YES | 生命周期主控器：`PENDING_RECEIVE, PENDING_TEST, IN_PROGRESS, OUTSOURCING, COMPLETED, CANCELLED` |
| `process_type` | ENUM | NO | 执行主体分级：`INTERNAL = 内部自建` / `EXTERNAL = 第三方委外` |
| `priority` | ENUM | YES | 缓急通道：`NORMAL`, `URGENT`, `CRITICAL` |
| `is_urgent` | BOOLEAN | YES | 核心告警阀：判断是否使用 4H 时效规则监控并报警加持 |
| `is_blind_test` | BOOLEAN | YES | 控制位：该检验员测试此批次是否启动去主观化的盲测面板屏蔽数据 |
| `sample_code` | VARCHAR(50) | NO | 系统分配实物标贴序列，也是 IoT 仪器识别数据的标识锚点 |
| `received_at` | DATETIME | NO | 样品录入并接收状态的时间戳 |
| `operator_id` | VARCHAR(30) | NO | 操作经办人对应 |

### 8.2 关联子表一：实验室结果明细表 (`qms_lab_result`)
**说明：** 挂载于具体的某一次/某一个主测试任务下，记录真正的物化检测数值及其产生的判断状态，供判定或报告引用。
| 字段名称 | 类型 | 是否必填 | 说明/用途 |
|----------|------|---------|------------|
| `id` | VARCHAR(50) | YES | 主键 |
| `task_no` | VARCHAR(50) | YES | 外键映射：依附到具体的主单据记录 |
| `data_type` | ENUM | YES | 区测指标值判断域：`QUANTITATIVE (定量型)` / `QUALITATIVE (定性型)` |
| `target_value` | DECIMAL | NO | 定量判定中心要求值 |
| `upper_limit` | DECIMAL | NO | 定量允许上边界约束 |
| `lower_limit` | DECIMAL | NO | 定量允许下边界约束 |
| `measured_value`| VARCHAR(200) | NO | 【核心实测记录区】(可手动更新或被 IoT 推送接口回调覆写数值) |
| `judgment` | ENUM | YES | 该行特征判定结果标定：`PASS / FAIL / CONDITIONAL` |
| `data_source` | ENUM | YES | 数据来源类型：标明本条数据系 `MANUAL` (手工敲击), `IOT_DEVICE` (仪器), 或 `EXTERNAL_REPORT` (第三方结果录入提取) |
| `is_critical` | BOOLEAN | YES | **控制锁控字段**：若设为 true，一旦测出 FAIL，调用外部仓储货架强制锁接口(WMS)及通知 |
| `phenomenon_code`| VARCHAR(50) | NO | 失效时的统一描述集外键指向 |
| `tested_at` | DATETIME | NO | 实测确认的具体时刻定格 |
| `rawDataLink` | VARCHAR | NO | 关联仪器/记录表的云链接或服务器物理相对路径 |

### 8.3 关联子表二：留样台账表 (`qms_lab_sample`)
**说明：** 根据主干业务完结后，通过抽检部分进行“保存待复查”管控所留底。
| 字段名称 | 类型 | 是否必填 | 说明/用途 |
|----------|------|---------|------------|
| `id` | VARCHAR(50) | YES | 日志台账序 |
| `sample_code` | VARCHAR(50) | YES | 映射的标识条码号 |
| `task_no` | VARCHAR(50) | YES | 来源依据任务标的 |
| `location` | VARCHAR(30) | YES | 实物柜体坐标（如 恒温库-1排3列）的配置录 |
| `retention_status`| ENUM | YES | 这个样本实物在台账上的流转声明：`RETAINED`(保存占用中)，`DISPOSED`(合规废除)，`RETURNED`(回退客户端或现场) |
| `deposit_date` | DATE | YES | 上架起始追踪日期 |
| `retention_date`| DATE | YES | **核心计算日**: `Deposit Date + 预设保留策略期`(到期被后端定时轮询提取扫出) |
| `handler` | VARCHAR(30) | NO | 管理建档人员或最后变更确认者的信息 |

### 8.4 关联子表三：实验室检测报告库 (`qms_lab_report`)
**说明：** 主干任务结束后的结果格式化呈现文件或外协文件的中心化留存系统库。
| 字段名称 | 类型 | 是否必填 | 说明/用途 |
|----------|------|---------|------------|
| `report_no` | VARCHAR(50) | YES | 流水报告系统标识码 |
| `task_no` | VARCHAR(50) | YES | 对应主干任务约束来源 |
| `file_url` | VARCHAR(1000) | NO | **核心应用域**: 电子化归档生成的附件 PDF 或者图库打包路径地址 |
| `judgment` | ENUM | YES | 主任务对结果的最后评估定性拷贝 `PASS/FAIL/CONDITIONAL` |
| `review_status` | ENUM | YES | 用于内部法权或合规：报告的确认签核机制 `DRAFT/PENDING_REVIEW/APPROVED/REJECTED` |
| `generated_by` | ENUM | YES | 形式认定：说明系 `AUTO`(取内测集通过特定文档套件渲染后形成)，或 `EXTERNAL`(人工扫描上传由外部合作公司 SGS 等提交的文件附件) |

## 10. 全生命周期与动作映射流转节点图示说明

### 10.1 主核实验室任务执行状态链条 (Main Flow)
> 表征从“来件申请”直至“出具合格证书/不合规结论”的主进程轨迹：
```
                   → (内部分析流程) → [PENDING_TEST (内测待办)]  --- 实验员选中进行操作
                  /                                              ↓
[PENDING_RECEIVE]                                               [IN_PROGRESS (检测行进中且录入数据)] 
 (需求到达等待接样)                                                ↓ (判断/生成对应报告并自动审批)
                  \                                              ↓
                   → (第三方委外流程) → [OUTSOURCING (交接发出物流)]   --> 第三方回传扫描并反馈结果
                                                                 ↓
                                                            [COMPLETED (任务终结及判定录入释放)]
```

### 10.2 日常留样监控状态链条 (Material Retention Flow)
> 主分析完成后（当符合需物理归档的试料标准：即进入该副流程控制），其针对样品物理留样管理的处置监控机制：
```
[生成入库并标识位置坐标、设置3个月限期]
       ↓
[RETAINED / 正在留样中]  =============> (常态维持期，期间不被干预系统挂起记录状态)
       ↓ (触发系统设定的检测期限)
   {到期警报: 定时脚本通知处理人}
       ↓
  （执行人的处置方向二选一） 
      ↙                      ↘
[DISPOSED / 物理销毁核销处理]     [RETURNED / 退回寄方或进入降级报废程序退客仓]
```

### 10.3 控制性文档及报告合法化签核流转链 (Report Compliance Flow)
> 独立控制报告法效承认流程图，用于报告的发布：
```
  (自动套件生成/人员手工建制上报文档)
                      ↓
              [DRAFT 草稿 / 本地域]
                      ↓ (发确认申请流)
           [PENDING_REVIEW 待特定管理岗签核审核] 
                      ↓ (高层介入或专家技术确认)
           ↙(发现问题退回重分析)    ↘(准确无误准予出图打印并发放)
[REJECTED/被拒接驳退]           [APPROVED / 已经审核(锁定文档只读状态提供引用查阅下载)] 
```
