# 受控文档管理模块 — 详细设计文档

> **文档版本**：V2.0
> **编写日期**：2026-03-02
> **适用范围**：舜富精密 S_QMS 1.0 — 受控文档管理（知识库与培训模块）
> **合规要求**：IATF 16949:2016 §7.5 文件化信息
> **接口依据**：《舜富PLM接口文档》4.5 文档元数据批量同步接口

---

## 1. 概述

### 1.1 模块目标

受控文档管理模块旨在为汽车零配件质量管理体系提供统一的文档存储、检索、版本控制和生命周期管理能力。该模块覆盖从质量手册、程序文件到工程图纸、检验报告在内的 20+ 种核心文档，满足 IATF 16949 中对"文件化信息"的受控要求。

核心特性：
- **业务场景导向的分类体系**：基于 5 大业务场景 × 11 个二级模块的二级分类树
- **复合存储模型**：支持 PLM（产品生命周期管理）和 QMS（质量管理系统）双源文档管理
- **差异化权限控制**：PLM 文档只读引用（查看/同步），QMS 文档全生命周期管理（新增/审批/发布/作废）
- **版本追溯**：文档版本历史记录、历史版本恢复
- **PLM 接口对齐**：数据模型与字段命名严格参照《舜富PLM接口文档》4.5 节标准

### 1.2 设计依据

| 来源 | 说明 |
|------|------|
| IATF 16949:2016 §7.5 | 文件化信息的创建、更新、控制要求 |
| 《舜富PLM接口文档》§4.5 | 文档元数据批量同步接口——字段名、枚举值、业务规则的权威标准 |
| 企业 PLM-QMS 协同需求 | PLM 主存文档在 QMS 中的引用与同步 |
| OpenSpec 提案 `add-controlled-document-management` | 初始需求——业务分类与混合存储 |
| OpenSpec 提案 `enhance-controlled-document-crud` | 增强需求——CRUD 页面与状态约束 |

## 2. 业务场景与文档分类

### 2.1 五大业务场景

系统将质量相关文档归入以下 5 大一级业务场景，每个场景下设若干二级模块：

| 编号 | 一级场景 (bizScene) | 含义 | 对应 IATF 章节 |
|------|---------------------|------|----------------|
| S1 | 体系策划与基础管理 | 质量手册、程序文件、标准法规 | §4.4, §7.5 |
| S2 | 产品设计与开发 (APQP) | 工程图纸、3D模型、BOM、FMEA、控制计划 | §8.3 |
| S3 | 生产过程与现场执行 | 作业指导书 (SOP/WI)、目视化标准 | §8.5 |
| S4 | 质量检验与监控记录 | IQC/IPQC/FQC/OQC 检验报告、实验报告 | §8.6, §9.1 |
| S5 | 异常管理与持续改进 | NCR报告、8D报告、审核报告 | §10.2, §10.3 |

## 4. 文档生命周期状态机

### 4.1 状态枚举对齐

> 状态枚举与 PLM 接口文档 4.5 节 `documentStatus` 保持一致。

| 状态码 | PLM接口原值 | QMS中文名 | 说明 |
|--------|-----------|-----------|------|
| DRAFT | DRAFT | 草稿 | 新建/驳回后的编辑态 |
| UNDER_REVISION | UNDER_REVISION | 修订中 | PLM文档修订中（QMS中按审批中处理） |
| RELEASED | RELEASED | 已发布 | 审批通过，正式生效 |
| OBSOLETE | OBSOLETE | 已作废 | 文档失效、被新版本替代 |

### 4.2 QMS 文档状态流转

```
                ┌──────────┐
    ┌──────────→│  DRAFT   │←──────────────┐
    │           │  (草稿)  │               │
    │           └────┬─────┘               │
    │                │ 提交审批             │ 审批驳回
    │                ▼                     │
    │           ┌──────────────────┐       │
    │           │ UNDER_REVISION   │───────┘
    │           │  (修订中/审批中)  │
    │           └────┬─────────────┘
    │                │ 审批通过
    │                ▼
    │           ┌──────────────┐
    │           │  RELEASED    │
    │           │  (已发布)    │
    │           └────┬─────────┘
    │                │ 作废
    │                ▼
    │           ┌──────────────┐
    └───────────│  OBSOLETE    │  （可基于此版本创建新草稿）
                │  (已作废)    │
                └──────────────┘
```

**状态转移规则**：

| 当前状态 | 允许操作 | 目标状态 | 前置条件 |
|----------|---------|---------|---------|
| DRAFT | 提交审批 | UNDER_REVISION | 必填字段已完善，附件已上传 |
| DRAFT | 删除 | (物理删除) | 仅草稿可删除 |
| DRAFT | 编辑 | DRAFT | 仅草稿可编辑 |
| UNDER_REVISION | 审批通过 | RELEASED | 审批人确认 |
| UNDER_REVISION | 审批驳回 | DRAFT | 审批人填写驳回意见 |
| RELEASED | 作废 | OBSOLETE | 填写作废原因 |
| OBSOLETE | 恢复版本 | DRAFT | 基于旧版本创建新草稿 |

### 4.3 PLM 文档状态约束

PLM 主存文档在 QMS 中的状态由 PLM 推送决定（通常为 `RELEASED`），不参与 QMS 内部审批流程。

| 操作 | PLM文档 | 说明 |
|------|---------|------|
| 查看 | 允许 | 查看文档详情、预览文件 |
| 下载 | 允许 | 通过 downloadUrl 或 QMS 代理下载 |
| 同步版本 | 允许 | 从PLM拉取最新版本 |
| 编辑 | 禁止 | 所有字段只读 |
| 删除 | 禁止 | 不可删除PLM引用（PLM推送 DELETE operation 时逻辑作废） |
| 状态流转 | 禁止 | 状态由PLM端通过同步接口控制 |

## 7. PLM 双向接口详细设计

### 7.1 接口概述

QMS 与 PLM 之间的文档同步严格遵循《舜富PLM接口文档》规范。核心接口为 **4.5 文档元数据批量同步接口**，采用 PLM 主动推送模式。

```
┌─────────┐                                          ┌─────────┐
│         │  ① 文档元数据批量同步                       │         │
│         │  POST /api/qm/plm/documents/metadata/     │         │
│   PLM   │  batch-sync                               │   QMS   │
│         │──────────────────────────────────────────→│         │
│         │                                          │         │
│         │  ② 质量数据反馈                            │         │
│         │←──────────────────────────────────────────│         │
│         │                                          │         │
│         │  ③ 工程变更推送 (4.6节)                    │         │
│         │──────────────────────────────────────────→│         │
└─────────┘                                          └─────────┘
```

### 7.2 接口认证与安全

> 遵循《舜富PLM接口文档》1.1 请求头规范。

| 请求头 | 说明 | 示例 |
|--------|------|------|
| X-API-Key | PLM系统分配的API密钥 | `plm_2024_ak_001` |
| X-API-Signature | HMAC-SHA256签名 | `sha256_hex_signature` |
| X-Request-ID | 请求追踪ID | `req_20240123_005` |
| X-Timestamp | 毫秒级时间戳 | `1705996800000` |
| Content-Type | 固定值 | `application/json` |

**签名生成规则**：参照《舜富PLM接口文档》第3章签名算法。

### 7.3 核心接口：文档元数据批量同步（PLM → QMS）

> 严格参照《舜富PLM接口文档》§4.5

| 项目 | 说明 |
|------|------|
| **接口功能** | PLM系统批量推送技术文档的元数据信息到QMS系统，建立文档关联关系 |
| **接口地址** | `POST /api/qm/plm/documents/metadata/batch-sync` |
| **调用方向** | PLM → QMS |
| **幂等性** | 支持（基于 documentId + documentVersion 的唯一性） |
| **批量大小** | 最大1000条/次 |
| **同步频率** | 文档发布/变更时实时触发或定时（每日凌晨）同步 |

#### 7.3.5 QMS 接收处理逻辑

```
接收 PLM 批量同步请求
       │
       ▼
  验证请求头签名
       │
       ▼
  遍历 documents 数组
       │
       ├── operation = CREATE
       │   ├── 校验 documentId+documentVersion 唯一
       │   ├── 校验关联产品/物料存在性
       │   ├── INSERT controlled_document
       │   │   (storage_source = 'PLM')
       │   ├── INSERT doc_version_history
       │   │   (source = 'PLM_SYNC')
       │   ├── 如有 downloadUrl → 异步下载文件
       │   │   → INSERT doc_attachment
       │   └── INSERT plm_sync_log (SUCCESS)
       │
       ├── operation = UPDATE
       │   ├── 查找已有记录 by document_id
       │   ├── 比较 documentVersion
       │   ├── UPDATE controlled_document
       │   ├── INSERT doc_version_history (新版本记录)
       │   ├── 如有文件变更 → 异步下载新文件
       │   └── INSERT plm_sync_log (SUCCESS)
       │
       └── operation = DELETE
           ├── 查找已有记录 by document_id
           ├── UPDATE document_status = 'OBSOLETE'
           ├── INSERT doc_version_history
           │   (change_summary = 'PLM端作废')
           └── INSERT plm_sync_log (SUCCESS)
       │
       ▼
  汇总处理结果，返回响应
```

#### 7.3.6 业务规则（参照PLM接口文档§4.5.5）

1. **唯一性校验**：documentId + documentVersion 组合必须唯一
2. **关联验证**：relatedProductCode 在 QMS 中需存在（如提供）
3. **状态流转**：documentStatus 变更需符合状态机规则
4. **日期有效性**：effectiveDate 不能晚于 expiryDate（如同时提供）
5. **CREATE操作**：documentId + documentVersion 组合必须唯一，必须提供所有必填字段
6. **UPDATE操作**：文档必须已存在于 QMS 中，不能更改 documentId 和 documentVersion
7. **DELETE操作**：实际执行逻辑作废（status 置为 OBSOLETE），被引用的文档不允许删除
8. **事务性**：单个文档失败不影响其他文档处理

#### 7.3.7 错误码（参照PLM接口文档§4.5.6）

| HTTP状态码 | 错误码 | 说明 | 处理建议 |
|-----------|--------|------|---------|
| 404 | RELATED_PRODUCT_NOT_FOUND | 关联产品不存在 | 先同步产品主数据或检查产品编码 |
| 404 | RELATED_MATERIAL_NOT_FOUND | 关联物料不存在 | 先同步物料主数据或检查物料编码 |
| 409 | DOCUMENT_DUPLICATE | 文档编号+版本重复 | 检查 documentId+documentVersion 唯一性 |
| 422 | VERSION_FORMAT_ERROR | 版本格式错误 | 检查版本号格式 |
| 422 | STATUS_TRANSITION_ERROR | 状态流转错误 | 检查文档状态变更是否合规 |

#### 7.3.8 调用频率限制（参照PLM接口文档§4.5.7）

| 环境 | 最大请求频率 | 最大批量大小 | 每日限额 |
|------|-----------|-----------|---------|
| 开发环境 | 10次/分钟 | 200条/次 | 10000条/天 |
| 测试环境 | 30次/分钟 | 500条/次 | 50000条/天 |
| 生产环境 | 60次/分钟 | 1000条/次 | 无限制 |

### 7.5 同步策略

#### 7.5.1 同步模式

| 模式 | 触发方式 | 范围 | 频率 | 说明 |
|------|---------|------|------|------|
| 全量同步 | 定时任务 | PLM全部受控文档 | 每日凌晨2:00 | 初始化或数据修复，PLM推送 operation=CREATE |
| 增量同步 | PLM事件推送 | 变更的文档 | 实时 | 常规运行模式，PLM推送 operation=CREATE/UPDATE/DELETE |
| 手动同步 | 用户操作 | 单个文档 | 按需 | 列表页"同步版本"按钮，QMS 主动调用PLM查询接口 |

#### 7.5.2 冲突处理策略

| 场景 | 处理方式 |
|------|---------|
| PLM documentVersion > QMS document_version | 正常更新，自动接收新版本 |
| PLM 推送 DELETE operation | QMS 中 document_status 置为 OBSOLETE，保留历史版本 |
| 同步失败（网络异常） | 记录 plm_sync_log（sync_status=FAILED），自动重试3次，间隔5/15/60分钟 |
| 同步失败（超过重试） | 发送告警通知至管理员，等待人工处理 |
| 文件下载超时 | 元数据已同步、附件标记为待下载，后台异步重试 |
| RELATED_PRODUCT_NOT_FOUND | 记录 plm_sync_log 含 error_code，提示需先同步产品主数据 |

#### 7.5.3 版本映射规则

```
PLM 文档版本更新流程：

PLM 推送 batch-sync
[{ operation: "UPDATE", documentId: "DRW-2024-001",
   documentVersion: "V3.1", documentStatus: "RELEASED", ... }]
                          │
                          ▼
              QMS 接收并验证签名
                          │
                          ▼
              查找 controlled_document
              WHERE document_id = 'DRW-2024-001'
                          │
                          ▼
              UPDATE controlled_document SET
                document_version = 'V3.1',
                document_status = 'RELEASED',
                plm_update_time = '...',
                checksum = '...',
                download_url = '...'
                          │
                          ▼
              INSERT doc_version_history
              (document_version='V3.1', source='PLM_SYNC')
                          │
                          ▼
              异步下载文件 → INSERT doc_attachment
                          │
                          ▼
              INSERT plm_sync_log (sync_status='SUCCESS')
                          │
                          ▼
              返回响应:
              successItems: [{ documentId: "DRW-2024-001",
                documentVersion: "V3.1", status: "UPDATED" }]
```

| 监控项 | 阈值 | 告警方式 |
|--------|------|---------|
