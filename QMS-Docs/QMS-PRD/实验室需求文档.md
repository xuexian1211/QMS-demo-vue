基于需求方案及前期设计的检验模型，**实验室送检（Laboratory Testing）**模块作为高精度、专项理化分析的核心，其设计旨在确保原材料及产品符合特定技术标准，并提供自动化的数据存证 。
以下是实验室送检详细需求、功能页面及业务流程：

1. 实验室送检核心业务需求
1.1 送检触发与分流 (Trigger & Dispatch)
● 多源触发：支持从 IQC（来料）、IPQC（过程）、OQC（出货）检验任务中，针对需精密仪器测定的项目（如材质分析、拉力测试）自动触发现场送检 。
● 手动任务：支持客诉分析或新产品研发（APQP 阶段）时，由工程师手动创建检测任务，并关联源头单据号 。
● 分流逻辑：系统需支持内部检测与外包检测（第三方）的分流管理 。
1.2 数据采集与自动化 (Data Collection & IoT)
● 设备互联 (IoT)：针对高频、大数据的设备（如三坐标 CMM、光谱仪、拉力机），优先实施设备联网，实现数据自动抓取，避免人工录入错误 。
● 盲测控制 (Blind Test)：在涉及多人对比或特定实验时，支持“盲测模式”，录入员无法查看其他人的历史数据，消除主观偏差 。
1.3 样品与报告管理 (Sample & Report)
● 留样管理：系统需记录留样位置与期限（如 3 个月），支持到期自动提醒处置，以便质量复查 。
● 报告自动生成：系统根据采集数据自动套用模板，对比标准值生成《检测/试验报告》，并自动判定合格状态 。
1.4 预警监控 (Alert System)
● 检测时效预警：常规任务 24 小时未出报告标记延误；加持“加急”标识的任务若超过 4 小时未出报告，通过钉钉强提醒实验室主任 。
● 结果异常预警：若检测出致命缺陷（如有害物质超标），立即锁定相关物料批次并弹窗报警给品质总监 。

2. 功能页面设计
页面名称	主要功能描述	关键字段/操作
送检任务工作台	实验室人员处理任务的核心入口，展示待处理、进行中任务 。	任务优先级、来源单据、加急状态。
实验室主数据配置	维护实验项目、检测标准、设备关联关系 。(已完成，参考检验项目)	实验类型、设备接口协议、判定标准。
内部实验登记	针对不具备自动采集条件的实验，进行人工录入 。	实验参数、环境温度/湿度、实测数值。
委外检测管理	记录发送至第三方（如 SGS）的进度与回传报告 。	委外供应商、快递单号、报告附件上传。
留样管理台账	监控样品库房状态 。	留样库位、存入日期、预定销毁日期。
检测/试验报告库	所有历史报告的查询、预览及打印中心 。	报告编号、判定结果、审批状态。

3. 业务流程设计
3.1 内部检测标准流程
1. 需求触发：IQC/IPQC 检验员在执行任务时，勾选“实验室送检”项，系统自动生成实验室任务 。
2. 样品接收：实验员扫码接收实物样品，系统状态更新为“检测中” 。
3. 执行与采集：
  ○ 自动：设备完成测量后，通过 IoT 接口将数据回传至 QMS 对应任务单 。
  ○ 手动：实验员在线录入实测值。
4. 报告生成与审批：系统匹配检验方案中的标准值，自动生成报告并触发内部审批流 。
5. 结果反馈：结果同步回源检验任务（如 IQC 任务单），并根据判定结论决定是否解锁库存或报废 。
3.2 委外检测协作流程
1. 委外申请：实验室主任判定内部无能力检测，发起委外申请 。
2. 采购协作：钉钉通知采购部对接第三方实验室，并在系统中记录委外单号 。
3. 报告录入：收到第三方纸质或电子报告后，由 AQE 将关键数据及附件上传至 QMS 。
4. 同步判定：系统根据上传数据同步更新关联业务单据的状态。
3.3 样品留样与处置流程
1. 入库：检测完成后，系统提示该物料是否需留样。若需，实验员记录库位 。
2. 预警：系统根据周期配置，在样品过期前发送提醒 。
3. 处置：实验员定期清理样品，并在系统中勾选“已销毁”或“退还”，确保台账实物一致。
