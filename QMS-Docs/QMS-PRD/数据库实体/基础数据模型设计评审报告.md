### 3.1 客户档案表（qm_bd_customer_archive）

#### ✅ 符合设计的地方
| 设计要求 | 数据库实现 | 符合性 |
|---|---|---|
| 多组织支持 | `org_id` 字段存在 | ✅ |
| 客户编码/名称 | `customer_code`, `customer_name` | ✅ |
| 客户等级 | `customer_grade` | ✅ |
| 信用等级 | `credit_level` | ✅ |
| 统一社会信用代码 | `social_credit_code` | ✅ |
| 联系方式 | `contact_phone`, `contact_email` | ✅ |
| 法定代表人 | `legal_representative` | ✅ |
| 银行信息 | `opening_bank`, `bank_account_number` | ✅ |
| 索引设计 | `idx_org_id`, `idx_customer_code`, `idx_customer_name`, `idx_customer_grade` | ✅ |

#### ⚠️ 需改进的地方
| 问题 | 现状 | 建议 | 优先级 |
|---|---|---|---|
| 缺少客户类型字段 | PRD设计中有`customerType`（国内/海外/内部交易） | 新增 `customer_type` VARCHAR(20) | 高 |
| 缺少联系人字段 | PRD设计中有`contactPerson` | 新增 `contact_person` VARCHAR(50) | 高 |
| 缺少联系地址 | PRD设计中有`address` | 已有`detailed_address`，需确认是否等价 | 中 |
| 缺少客户代码索引唯一性约束 | 同一组织下客户编码应唯一 | 新增唯一索引 `uk_org_customer_code(org_id, customer_code)` | 高 |
| 缺少供应商档案表 | 设计文档中有供应商档案模块 | **SQL脚本中未包含供应商档案表** | 高 |

#### 📋 改进建议SQL
```sql
-- 新增客户类型字段
ALTER TABLE `qm_bd_customer_archive` 
ADD COLUMN `customer_type` VARCHAR(20) NULL COMMENT '客户类型（DOMESTIC-国内/OVERSEAS-海外/INTERNAL-内部交易）' AFTER `customer_grade`;

-- 新增联系人字段
ALTER TABLE `qm_bd_customer_archive` 
ADD COLUMN `contact_person` VARCHAR(50) NULL COMMENT '联系人' AFTER `contact_email`;

-- 新增组织+客户编码唯一索引
ALTER TABLE `qm_bd_customer_archive` 
ADD UNIQUE INDEX `uk_org_customer_code`(`org_id` ASC, `customer_code` ASC) USING BTREE;
```
