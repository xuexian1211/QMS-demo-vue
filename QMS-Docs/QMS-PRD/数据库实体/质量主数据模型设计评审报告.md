## 三、逐表详细评审

### 3.1 检验项目标准库（qm_md_insp_item）

#### ✅ 符合设计的地方

| PRD设计要求 | 数据库实现 | 符合性 |
|---|---|---|
| 项目编码唯一 | `code` + `uk_org_code(org_id, code)` | ✅ |
| 项目名称 | `name` VARCHAR(255) | ✅ |
| 归属组织（集团级支持） | `org_id` NULL表示集团级 | ✅ |
| 分类（尺寸/外观/理化/功能） | `category` VARCHAR(100) | ✅ |
| 数据类型（计量型/计数型） | `data_type` VARCHAR(30) | ✅ |
| 单位CODE | `unit_code` VARCHAR(30) | ✅ |
| 检验方法关联 | `insp_method_id` → qm_md_insp_method | ✅ |
| 量具类型关联 | `measure_type_id` → qm_md_measure_type | ✅ |
| 实验室标识 | `enable_lab` INT | ✅ |
| SPC标识 | `enable_spc` INT | ✅ |
| 索引设计 | org_id, code, category, data_type | ✅ |

#### ⚠️ 需改进的地方

| 问题 | 建议 | 优先级 |
|---|---|---|
| 缺少特殊特性标识字段 | 新增 `special_characteristic` VARCHAR(20) 标识CC/SC/一般特性 | 🔴 高 |
| 缺少检验项状态字段 | 新增 `status` VARCHAR(20) 管理生命周期 | 🟡 中 |
| 缺少默认规格值字段 | 新增 `default_usl`, `default_lsl`, `default_target` DECIMAL | 🟡 中 |

#### 📋 改进建议SQL

```sql
ALTER TABLE `qm_md_insp_item` 
ADD COLUMN `special_characteristic` VARCHAR(20) NULL COMMENT '特殊特性（CC-关键特性/SC-重要特性/NC-一般特性）' AFTER `measure_type_id`,
ADD COLUMN `status` VARCHAR(20) DEFAULT 'ACTIVE' COMMENT '状态（ACTIVE-生效/INACTIVE-停用/OBSOLETE-作废）' AFTER `special_characteristic`,
ADD COLUMN `default_usl` DECIMAL(18,6) NULL COMMENT '默认规格上限' AFTER `status`,
ADD COLUMN `default_target` DECIMAL(18,6) NULL COMMENT '默认目标值' AFTER `default_usl`,
ADD COLUMN `default_lsl` DECIMAL(18,6) NULL COMMENT '默认规格下限' AFTER `default_target`;
```

## 八、评审结论

### 8.1 总体评价

本次评审的 `qm_bd_customer_archive.sql` 中的**质量主数据模型（qm_md_前缀）**整体设计**优秀**，在以下方面表现突出：

**✅ 优点**：
- 命名规范统一，符合团队开发规范
- 支持多组织（`org_id`）和多租户（`f_tenant_id`）架构，集团级与工厂级数据隔离设计完善
- 审计字段完整，便于追溯
- 检验模板的版本管理和状态流转设计完善
- 抽样方案的三级树状体系（Plan→Rule→Detail）设计专业
- 不良现象与原因的关联知识库设计支持智能推荐
- 索引设计合理，覆盖常用查询字段

**⚠️ 需改进**：
- 部分表主键类型不一致（BIGINT vs VARCHAR）
- 检验计划/方案主表缺失
- 量检具台账缺少校准周期等关键字段
- 部分表缺少状态/版本管理字段
